# HIP.md — Human Interface Protocol

## Human Operating Mode

Role in this repo: Student / learner
Technical fluency: Variable; actively learning the stack; some coding background, limited experience with Git, terminal, and deployment
Primary intent: Learn the stack
Autonomy level: Small safe edits only
Explanation depth: Teach as we go
Risk tolerance: Conservative
Decision style: Tradeoffs then recommendation
Feedback style: Patient teacher

## Technical Fluency

| Domain | Level |
|--------|-------|
| Code | Medium |
| Git | Low |
| Terminal | Low |
| Debugging | Low |
| Deployment | Not applicable |
| Architecture | Low |
| Database | Low |
| Dependencies | Low |

## Approval Required Before

- authentication
- database schema
- environment variables
- deployment configuration
- migrations
- any change that cannot be easily undone

## Review Workflow

- explain what you are about to do before doing it
- explain what changed and why after each step
- point out where to look in the code to understand what happened
- ask if I want to try the next step myself before proceeding

## Definition of Done

Work is not done until:

- the change works and I understand why
- I can explain what changed to someone else
- next steps are clear
- I know what to read or try next to keep learning

## Update Rule

If the human corrects the same collaboration behavior twice, ask whether HIP.md should be updated.
