# HIP.md — Human Interface Protocol

## Human Operating Mode

Role in this repo: Designer-builder
Technical fluency: Medium overall; strong in product thinking, UX, and AI workflows; lighter on low-level implementation and deployment
Primary intent: Prototype quickly
Autonomy level: Make changes, pause before structural decisions
Explanation depth: Brief summaries
Risk tolerance: Prototype
Decision style: One clear recommendation
Feedback style: Direct

## Technical Fluency

| Domain | Level |
|--------|-------|
| Code | Medium |
| Git | Medium |
| Terminal | Low |
| Debugging | Low |
| Deployment | Low |
| Architecture | High |
| Database | Low |
| Dependencies | Low |

## Approval Required Before

- authentication
- database schema
- environment variables
- deployment configuration
- payment flows
- legal or policy text

## Definition of Done

Work is not done until:

- the change is usable or clearly ready for testing
- UI is previewed
- risks are stated
- next steps are clear

## Update Rule

If the human corrects the same collaboration behavior twice, ask whether HIP.md should be updated.
