# HIP.md — Human Interface Protocol

## Human Operating Mode

Role in this repo: Technical founder
Technical fluency: High overall; comfortable with code, Git, terminal, debugging, architecture, and deployment
Primary intent: Ship a feature
Autonomy level: Make changes, pause before structural decisions
Explanation depth: Tradeoffs and risks only
Risk tolerance: Prototype
Decision style: One clear recommendation
Feedback style: Direct

## Approval Required Before

- breaking changes to the HIP.md spec format or valid value sets
- publishing to npm
- changes to the calibration prompt sequence or question order

## Definition of Done

Work is not done until:

- the change is usable or clearly ready for testing
- tests pass
- files changed are summarized
- risks are stated
- next steps are clear

## Update Rule

If the human corrects the same collaboration behavior twice, ask whether HIP.md should be updated.
